## 概述

### 1. 总体目标和范围

本计划用于把 `md-reader` 从“本地服务只读工作区”推进到“本地服务可编辑、可保存、可恢复状态的 Markdown 工作区”。本阶段只保留 `local-service` 正式运行形态，直接移除浏览器授权模式的产品分支；首阶段验收目标聚焦于读取、导航、搜索、状态恢复继续稳定可用，并新增编辑、保存、冲突保护、未保存修改确认和 `split` 工作区能力。

### 2. 各阶段任务概要

- 阶段 A：收敛运行边界。移除 browser 正式分支，统一前端数据源与状态机。
- 阶段 B：补齐文档读写接口。先打通 `mtimeMs/size` 读取元数据，再新增写回与冲突校验接口。
- 阶段 C：接入编辑工作区。补 `edit` / `split` 渲染、编辑状态、保存栏和快捷键。
- 阶段 D：补未保存保护。统一处理切文档、切项目、切 profile、重启/关闭服务和离开页面。
- 阶段 E：回归验证。覆盖单测、运行器、实际本地服务读写链路与状态恢复。

### 3. 整体结构框架

- `server/`：负责 Markdown 读取元数据、写回、mtime 冲突保护和本地服务 API。
- `src/workspace/`：负责本地服务访问、项目/状态持久化和工作区数据入口。
- `src/app/`：负责三栏工作区、编辑区、预览区、标题导航和未保存确认流。
- `src/editor/`：负责原始 Markdown 编辑器和保存状态条。
- `tests/`：按服务端、工作区访问层、UI 状态流分层补测试。

## 执行顺序

### 阶段 A：收敛为本地服务唯一主链路

目标：
- 移除 `showDirectoryPicker()` 与 browser 授权恢复的正式产品逻辑。
- 将 `workspaceSource` 收敛为 `local-service | offline`。
- 保留现有 `open/stop/health` 与服务控制按钮。

主要修改点：
- `src/App.tsx`
- `src/workspace/workspace-provider.ts`
- `src/workspace/registry.ts`
- `src/workspace/browser-project-access.ts`
- 相关测试

验收：
- 本地服务在线时可以正常恢复项目、文件树和已读文档。
- 本地服务离线时进入 `offline` 提示，不再出现 browser fallback。

### 阶段 B：补文档读取元数据与写回接口

目标：
- `GET /document` 返回 `content + mtimeMs + size`。
- 新增 `POST /document` 写回接口，支持 `expectedMtimeMs` 乐观并发保护。
- 写回成功后返回新的 `mtimeMs` 和 `size`。

主要修改点：
- `server/project-reader.mjs`
- `server/bridge-server.mjs`
- 新增 `server/project-writer.mjs`
- `src/workspace/local-bridge-access.ts`
- 相关测试

验收：
- 前端能拿到稳定的文档版本锚点。
- 磁盘文件被外部改动后，保存请求返回冲突错误而不是静默覆盖。

### 阶段 C：接入编辑模式与 split 工作区

目标：
- `edit` 模式显示原始 Markdown 文本编辑。
- `split` 模式采用中间工作区内部左右分栏，右侧标题导航继续独立保留。
- 阅读态使用 `savedContent`，编辑态和分栏预览使用 `editingContent`。

主要修改点：
- `src/App.tsx`
- `src/app/AppShell.tsx`
- `src/app/WorkspaceLayout.tsx`
- 新增 `src/editor/raw-markdown-editor.tsx`
- 新增 `src/editor/document-save-bar.tsx`
- 相关测试

验收：
- 可以切到编辑态输入文本。
- 可以在分栏模式下左侧编辑、右侧实时预览。
- 标题导航基于当前编辑内容实时更新。

### 阶段 D：补未保存修改保护与状态恢复

目标：
- 未保存修改时，切文档、切项目、切 profile、重启服务、关闭服务、刷新/离开页面统一弹出三选确认。
- 三选项固定为“保存 / 不保存 / 取消”。
- 状态恢复继续保存上次打开项目、当前文档、模式、滚动位置和阅读进度；项目记录继续写入个人配置，本地阅读状态继续只存本地。

主要修改点：
- `src/App.tsx`
- `src/workspace/local-state.ts`
- `src/workspace/profile-store.ts`
- 相关测试

验收：
- 有脏状态时不会无提示丢内容。
- 选择“保存”后继续原动作；选择“不保存”丢弃并继续；选择“取消”停留原位。

### 阶段 E：回归验证

目标：
- 保证本地服务模式下的主链路完整可用。

验证项：
- `npm test`
- `npm run health`
- `npm run open -- --project C:\\Code\\Nocturnel`
- 实际打开文档、编辑、保存、切文档、重启服务、恢复状态

## 本阶段实现策略

1. 先测后改，每个阶段先补失败测试，再做实现。
2. 先改服务契约，再改前端状态，避免前端先写成猜测接口。
3. 先移除 browser 正式分支，再接编辑保存，避免双状态机并行增加复杂度。
4. 首阶段不做兼容过渡；旧逻辑直接清理，只保留本地服务新链路。

## 本阶段验收标准

- 本地服务是唯一正式主链路。
- Markdown 文档可以读取、编辑、保存并做 `mtime` 冲突保护。
- `read / edit / split` 三种模式行为符合方案定义。
- 右侧标题导航在阅读态和编辑态都稳定工作。
- 未保存修改保护覆盖文档切换、项目切换、profile 切换、服务操作和页面离开。
