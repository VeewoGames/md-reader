# 左侧文件树 Codex 风格实施计划

## 概述

### 1. 总体目标和范围

本计划用于落实左侧文件树的 Codex 风格调整：让目录和文件结构更接近导航树，而不是按钮列表，并补上目录折叠/展开与当前路径自动展开能力。范围限定在左侧文件树组件和对应样式，不改数据源、不改文档读取流程。

### 2. 各阶段任务概要

1. 组件交互改造
   主要工作：在文件树组件内部引入目录展开状态、目录点击切换、当前路径祖先自动展开。
   预期成果：文件树从静态递归渲染改为可展开树。
   执行顺序：先做。

2. 样式重构
   主要工作：重写文件树相关 CSS，使目录和文件行呈现更轻的 Codex 风格树形导航。
   预期成果：左侧树视觉明显摆脱按钮块外观。
   执行顺序：组件交互后执行。

3. 验证
   主要工作：运行构建或静态检查，确认改动不破坏现有 React/TypeScript 代码。
   预期成果：行为和构建结果可接受。
   执行顺序：实现完成后执行。

### 3. 整体结构框架

- `src/app/WorkspaceLayout.tsx`
  - 管理目录展开状态
  - 递归渲染目录和文件节点
  - 根据 `currentDocumentPath` 派生需要展开的祖先目录
- `src/index.css`
  - 负责目录/文件行、箭头、导线、选中态和 hover 态

## 任务拆分

### 任务 1：文件树交互结构调整

**修改文件**

- `src/app/WorkspaceLayout.tsx`

**主要工作**

1. 为目录节点渲染独立按钮，而不是纯文本容器。
2. 在组件内部维护 `expandedDirectories`。
3. 计算当前文档路径的祖先目录集合，并在渲染时保证这些目录展开。
4. 仅在目录展开时渲染子树。

### 任务 2：文件树样式重构

**修改文件**

- `src/index.css`

**主要工作**

1. 重写 `.file-tree`、`.file-tree__item`、`.file-tree__directory`、`.file-tree__file`。
2. 新增目录按钮、箭头、子树容器、导向线等样式类。
3. 调整当前文件高亮，使其更像导航选中态。

### 任务 3：验证

**验证命令**

```powershell
npm run build
```

**验证目标**

1. TypeScript 与 Vite 构建通过。
2. 没有因为新增状态和 JSX 结构导致编译错误。
3. 左侧树功能范围内的改动没有影响其他区域的静态结构。
