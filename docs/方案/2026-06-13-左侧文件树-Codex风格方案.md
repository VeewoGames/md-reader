# 左侧文件树 Codex 风格方案

## 概述

### 1. 总体目标和范围

本轮目标是让 `md-reader` 左侧文件夹与文件结构更接近 Codex 的视觉与层级表达，同时保持当前文档选择能力不变。范围限定在左侧文件树区域的表现层与轻交互：目录/文件行样式、层级缩进、目录箭头、选中态、hover 态，以及目录折叠/展开行为。

本轮不包含文件树数据结构重写、不引入拖拽/重命名/右键菜单，也不改变项目接入、文档读取、右侧大纲和中间文档工作区逻辑。

### 2. 各阶段任务概要

1. 现状梳理
   主要工作：确认文件树组件、数据结构和样式边界。
   预期成果：明确只需改 `WorkspaceLayout.tsx` 与 `index.css`，不触碰 `buildFileTree` 输出协议。
   执行顺序：先做。

2. 交互与视觉收敛
   主要工作：定义接近 Codex 的目录/文件结构行、目录折叠行为、当前文档路径默认展开策略。
   预期成果：形成可执行的 UI 基线，不在实现中临时改方向。
   执行顺序：现状梳理后执行。

3. 实现与验证
   主要工作：落地组件状态与样式，验证目录展开、文件点击、当前文档高亮与现有功能兼容。
   预期成果：左侧树同时具备 Codex 风格层级表达和可用的基础交互。
   执行顺序：方案确认后执行。

### 3. 整体结构框架

- 数据层继续使用 `src/workspace/file-tree.ts` 生成的 `FileTreeNode[]`
- 组件层在 `src/app/WorkspaceLayout.tsx` 内部管理目录展开状态
- 样式层集中在 `src/index.css`，通过更轻的树形列表样式替换当前按钮卡片感

## 当前问题

当前左侧树存在以下偏差：

1. 文件项是按钮卡片样式，视觉重量偏高，更像操作按钮而不是导航树节点。
2. 目录只是纯文本，没有箭头、展开态、点击反馈，也没有树形控件的结构感。
3. 层级主要依赖 `ul` 缩进，缺少像 Codex 那样的细导线、稳定行高和凑近内容区的阅读节奏。
4. 当前文档高亮偏接近“按钮选中”，不够像侧边导航里的当前项。

## 目标体验

参考 Codex 左侧树，本轮对齐以下体验：

1. 目录是可点击的树节点，带小箭头并支持折叠/展开。
2. 文件是轻量文本行，hover 与当前项高亮都更克制。
3. 层级缩进一致，子节点有细导向线，便于连续扫描。
4. 打开文档时，其祖先目录默认展开，避免当前文档落在折叠目录里。

## 推荐方案

采用“样式 + 轻交互”的 B 方案。

### 视觉策略

1. 左侧树从“按钮块”改成“导航树行”：
   - 目录行与文件行统一为低高度、紧凑布局。
   - 取消文件按钮的厚边框和独立卡片感。
   - 以轻背景 hover 和更窄的圆角作为反馈。

2. 层级表达改为 Codex 风格近似：
   - 每一层子树使用统一缩进。
   - 子树区域补一条轻量竖向引导线。
   - 目录箭头使用纯 CSS/字符旋转，不额外引入图标依赖。

3. 当前项高亮采用导航态而非按钮态：
   - 使用较浅底色与轻边界强调当前文件。
   - 颜色仍留在现有暖色主题范围内，避免突然切成深色 Codex 皮肤。

### 交互策略

1. 目录节点支持点击展开/折叠。
2. 默认展开所有根目录，避免首次进入时树为空白。
3. 当前文档路径上的祖先目录强制展开。
4. 文件节点仍然只做文档选择，不新增其他动作。

## 影响文件

- `src/app/WorkspaceLayout.tsx`
  - 为目录节点补展开状态、切换逻辑、当前路径祖先展开逻辑。
- `src/index.css`
  - 重写文件树区域样式，使其从按钮列表过渡为树形导航。

## 非目标

以下内容不在本轮范围：

1. 文件树虚拟滚动
2. 拖拽排序
3. 右键菜单
4. 文件/目录图标体系
5. 持久化目录展开状态
6. 修改 `FileTreeNode` 数据结构或 `buildFileTree` 构建逻辑

## 验收标准

1. 左侧目录可展开/折叠，且不会影响文件点击打开。
2. 当前文档所在路径在渲染时会自动展开。
3. 文件树视觉上明显更接近 Codex 的左侧导航树，而不是按钮堆叠。
4. 中间文档阅读、顶部工具栏、右侧占位区行为保持不变。
