# md-reader 移除阅读层与锁定态主编辑实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 将 `md-reader` 的主工作区从 `read / edit / split` 收敛为 `常规 / 分栏`，让 `常规` 默认以锁定态打开、解锁后原地编辑，并保持 `split` 作为源码精修模式。

**Architecture:** 第一阶段就要求 `常规` 锁定态和解锁态共用同一套 `VisualMarkdownEditor` 骨架，只通过 `readonly / editable` 切换工作态；不接受锁定态继续复用 `ReadonlyMarkdownRenderer`。执行顺序应先给编辑器入口增加只读能力，再让 `WorkspaceLayout` 在 `regular` 下无条件走同一编辑器入口，最后再把锁定/解锁与现有 autosave flush 联动。

**Tech Stack:** React 19、TypeScript、Vite、Vitest、Testing Library、Milkdown、Crepe、本地 bridge API

---

## 概述

### 1. 总体目标和范围

- 总体目标：
  - 顶部模式从 `阅读 / 编辑 / 分栏` 收敛为 `常规 / 分栏`
  - 默认进入 `常规 + 锁定态`
  - 解锁后原地编辑，不产生“切到另一页”的明显感觉
  - `split` 左侧源码编辑完全不受锁定按钮影响
- 本阶段范围：
  - 改写模式类型与状态模型
  - 调整 `TopBar` 文案与锁定按钮
  - 调整 `WorkspaceLayout` 主分支
  - 将锁定动作接入现有 `flushActiveDraft`
  - 明确“解锁是全局工作区状态，不跟随文档”
  - 补齐组件、状态机、集成和浏览器回归测试
- 不在本阶段范围：
  - 删除所有只读技术实现文件
  - 重写 `split` 左侧源码编辑器
  - 表格、callout、frontmatter、HTML block 等复杂结构能力扩展
  - 重写 autosave 状态机

### 2. 各阶段任务概要

- 阶段 A：编辑器只读能力接入
  - 主要工作：给 `VisualMarkdownEditor` 增加 `readonly` 能力，并验证锁定态与解锁态共享同一骨架
  - 预期成果：`regular` 的锁定与解锁不再依赖两套组件入口
- 阶段 B：模式与状态模型收敛
  - 主要工作：把 `WorkspaceMode` 改成 `regular | split`，新增 `regularViewState`
  - 预期成果：状态层不再存在 `read`
- 阶段 C：顶部与工作区交互收敛
  - 主要工作：替换顶部模式按钮，新增锁定按钮，把 `WorkspaceLayout` 改成 `regular / split`
  - 预期成果：用户只看到 `常规 / 分栏`
- 阶段 D：锁定/解锁与 autosave 联动
  - 主要工作：把“锁定”动作接入现有 `flushActiveDraft`
  - 预期成果：保存成功后再锁定，失败则保持解锁
- 阶段 E：测试与浏览器回归
  - 主要工作：补齐状态机、组件、集成、浏览器回归
  - 预期成果：新模式稳定，不回退 `split`、标题导航、autosave

### 3. 整体结构框架

- 顶层模式：
  - `regular`
  - `split`
- 常规页视图状态：
  - `locked`
  - `unlocking`
  - `editable`
  - `locking`
- 精修页：
  - 左侧源码编辑
  - 右侧预览

- [ ] **Step 3: 在 `App.tsx` 中实现切换函数**

新增：

```ts
async function handleToggleRegularLock() {
  if (mode !== 'regular') {
    return
  }

  if (regularViewStateRef.current === 'locked') {
    setRegularViewState('unlocking')
    setRegularViewState('editable')
    return
  }

  if (regularViewStateRef.current !== 'editable') {
    return
  }

  setRegularViewState('locking')

  if (!(await flushActiveDraft())) {
    setRegularViewState('editable')
    return
  }

  setRegularViewState('locked')
}
```

注意：

- `unlocking` / `locking` 必须进入按钮禁用态
- `split` 下点击锁定按钮不会发生，因为按钮不会渲染

- [ ] **Step 4: 让切模式与切文档遵守全局解锁规则**

在 `handleDocumentSelect(path)` 中改为：

## 验收口径

- UI 层：
  - 顶部只显示 `常规 / 分栏`
  - `常规` 默认锁定
  - 锁定按钮具备 `locked / unlocking / editable / locking` 四态文案
- 骨架层：
  - `regular + locked` 与 `regular + editable` 共用同一个 `VisualMarkdownEditor` 组件入口
  - 锁定态不得继续走独立 `ReadonlyMarkdownRenderer` 主分支
- 状态层：
  - `mode` 只剩 `regular | split`
  - `regularViewState` 成为常规页唯一视图状态
- 行为层：
  - 解锁是全局工作区状态
  - 切文档时回到 `常规`，但保留当前锁定/解锁状态
  - 锁定动作必须先 flush 保存
- 保底边界：
  - `split` 左侧源码精修不回退
  - 标题导航继续稳定可用
  - autosave 主链不回退
