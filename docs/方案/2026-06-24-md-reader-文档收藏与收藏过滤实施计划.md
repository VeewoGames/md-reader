# 文档收藏与收藏过滤 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为 `md-reader` 增加文档级收藏、搜索框左侧“只看收藏”开关，以及“显示隐藏项 + 只看收藏”的正式组合模式，同时保证 profile 持久化、目录上下文、展开状态和当前文档链路稳定。

**Architecture:** 本轮实现沿用现有“原始树 + 派生渲染树 + 会话态开关”的工作区结构：`profile.navigation.favoritePaths` 保存长期收藏状态，`showFavoritesOnly` 只保存在 `App` 的会话内 React state，`src/workspace/file-tree.ts` 在隐藏可见性派生之后再叠加收藏过滤和搜索过滤。`WorkspaceLayout` 负责文件树局部工具条、文档行收藏按钮、零收藏空状态和组合模式渲染，不把收藏过滤抬升到 `TopBar`。

**Tech Stack:** React 19, TypeScript, Vitest, Testing Library, plain CSS in `src/index.css`

- [ ] **Step 5: 实现收藏过滤纯函数**

```ts
export function filterFileTreeByFavorites(
  nodes: VisibleFileTreeNode[],
  favoritePaths: string[],
): VisibleFileTreeNode[] {
  const favoritePathSet = new Set(favoritePaths)

  function visit(node: VisibleFileTreeNode): VisibleFileTreeNode | null {
    if (node.kind === 'file') {
      return favoritePathSet.has(node.path) ? node : null
    }

    const nextChildren = node.children
      .map(visit)
      .filter((child): child is VisibleFileTreeNode => child != null)

    if (nextChildren.length === 0) {
      return null
    }

    return {
      ...node,
      children: nextChildren,
    }
  }

  return nodes
    .map(visit)
    .filter((node): node is VisibleFileTreeNode => node != null)
}
```
